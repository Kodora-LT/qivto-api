# Contributing

Thank you for improving QIVTO.

1. Open an issue for a bug or a proposed API change.
2. Keep public API changes backward compatible whenever possible.
3. Never commit credentials, personal data or production configuration.
4. Add or update tests for client behaviour and error responses.
5. Run the language-specific test command documented in the client README.
6. Use a focused pull request with a clear description and verification steps.

Generated code must not replace reviewed client code without an explanation of the generator and reproducible output.
